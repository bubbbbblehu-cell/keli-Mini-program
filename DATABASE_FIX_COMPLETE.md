### 步骤 3：检查数据库（如果显示为空）

#### 3.1 访问 Supabase Dashboard
1. 打开 https://app.supabase.com
2. 选择项目（ID: `hmmruoankhohowlzajll`）
3. 进入 **Table Editor**

#### 3.2 检查 hotels 表
- 查看是否有数据
- 确认 `is_active` 字段为 `true`
- 确认 `safety_score` 字段有值

#### 3.3 如果没有数据，导入数据

**方式 1：手动添加（测试用）**
在 Table Editor 中点击 **Insert row**，填写：
- `name`: 酒店名称
- `address`: 地址
- `latitude`: 纬度（如 22.0096）
- `longitude`: 经度（如 100.7975）
- `safety_score`: 安全评分（0-5，如 4.5）
- `review_count`: 评价数（如 120）
- `is_active`: true
- `city_id`: 城市UUID（需要先在 cities 表中创建城市）

**方式 2：使用 SQL 批量导入**
参考 `DATABASE_TROUBLESHOOTING.md` 中的 SQL 示例

## 🐛 常见问题

### 问题 1：仍然显示"使用模拟数据"
**原因**：数据库中没有酒店数据
**解决**：在 Supabase Dashboard 中导入酒店数据

### 问题 2：显示"数据库连接失败"
**原因**：API 密钥错误或网络问题
**解决**：
1. 检查 `src/config/supabase.js` 中的密钥
2. 检查网络连接
3. 查看控制台的详细错误信息

### 问题 3：显示"找到 0 个匹配城市代码的酒店"
**原因**：数据库中的城市代码与应用不匹配
**解决**：应用现在会显示所有酒店，不受影响
